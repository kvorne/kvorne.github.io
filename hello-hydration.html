Overview

Assignment

Design an app for the Apple Watch that helps users manage an everyday task.

Context

This is an introductory project for Visual Communication Design 3, a course exploring web-based interactivity for desktop, mobile, and emerging digital contexts. The goal of this project is to practice simple, yet effective ux design by embracing the Apple Watch platform.

Scope

Design user flow and low-fidelity mockup. Conduct usability test and use feedback to design High-Fidelity prototype.

Timeline

3 Weeks
  • 1 Week discovering shared everyday tasks and hydration
  • 1 Week developing userflow and low-fidelity prototype
  • 1 Week user testing and designing high-fidelity prototype

My Role

I worked independently on this project guided by valuable feedback, user testing, and critiques from my professor and classmates. I acted as a UX Designer and Researcher throughout this project.

Discover

Finding Common Experiences

In order to brainstorm useful applications, our class collaborated to identify our daily goals and needs.

Zooming In

Exploring Apple Watch

Beyond learning about people's hydration habits and needs, I also took this project as an opportunity to familiarize myself with the Apple Watch interface. More specifically, I learned about user demographics and Apple Watch features.

The Apple Watch has a much more niche user base than ubiquitous platforms like desktop computers and mobile devices, therefore, I looked to the internet to find more information about user demographics andbest design practices.

User Demographics and Behaviors

There are important trends among Apple Watch Users:

  • A majority of users use the Apple Watch to track health and fitness.
  • Many users also use the app to check messages and manage reminders.
  • Most users check their watch 2-5 times an hour.
Key Features

The Apple Watch has a unique set of developer tools and user affordances:

  • Health kit hosting all of users' medical information
  • Health and Motion Sensors
  • Touch Screen
  • Watch Dial
  • Smart Stack Widgets

Define

Defining the Target User

Defining the User Need

Hydration-conscious Apple Watch users need a way to simply and efficiently track their daily water intake compared to their hydration goals.

Design

User Journey

To address the user need, I developed a User Journey Map that addresses the User Story: "As a hydration-consious individual, I need to easily efficiently track my daily water intake compared to my hydration goals."

Low-Fidelity Prototyping

The screens of the low-fidelity prototype follow the same sequence as the user journey map with the exception of the track progress page, as I decided that would add too much complexity for the Minimum Viable Product of a hydration app.

User Testing

Procedure

I ran usability testing with the low-fidelity prototype to validate the user flow before moving on to high-fidelity prototyping.

In this test, two users with no prior experience with the interface were given a link to the figma prototype and the task: "As a sleep deprived ND student during midterms, I want to track my hydration and add a Tall Latte to my daily consumption."

I observed these users navigate the platform and listened to their verbal and written feedback.

Findings

Both participants were able to successfully complete the task in the minimum number of clicks.

Some notable feedback I recieved included:

  • "Unclear what health kit is."
  • "I like how simple but useful the platform is."
  • "The buttons are too small for the watch face."
  • "I could see athletes using this."

This feedback led to the insight that the platform is appealing, but intimidating.

Based on user feedback and insights, I suggested the following revisions:

  • Use descriptive help text.
  • Enlarge text and buttons.
  • Use sporty, but welcoming branding to meet users wherever they are in their health journey.

Deliver

Features

Smart Stack Widget

The Apple Watch includes a "Smart Stack" of widgets that allow users to quickly browse timely app-based information. Hello Hydration makes use of this by allowing users to quickly compare their current water intake to their daily goal.

Timely Notifications

Additionally, the watch allows for full-screen notifications for important information. Users who take their hydration very seriously can opt into full screen notifications that indicate when they have become far below pace with their hydration goals.

Automatic Drinking Detection

Finally, the most important feature of this app is its timely notification that appears when the app detects a user finishing a drink so that they never forget to count their consumption! This feature makes use of the Apple Watch's unique motion detection capabilities to learn the cadence of the user's last sips of a drink.

High-Fidelity Prototyping

The screens of the high-fidelity prototype follow the same sequence as the low-fidelity screens, revised to read and look more inviting and motivational.
